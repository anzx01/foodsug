# **糖尿病患者食物升糖指数识别 H5 应用需求文档**

---

## **1. 项目概述**

### **1.1 项目名称**
糖尿病患者食物升糖指数识别 H5 应用（以下简称“**糖识 H5**”）

### **1.2 项目目标**
为糖尿病患者提供便捷的食物升糖指数（GI）识别工具，通过拍照或相册上传，结合 AI 视觉模型快速识别食物并返回 GI 值及健康建议；同时支持食物库筛选、血糖历史记录，帮助用户科学管理饮食与血糖。

### **1.3 目标用户**
- 确诊糖尿病患者（I型、II型、妊娠糖尿病）
- 关注血糖控制的亚健康人群
- 营养师/医生辅助工具用户

---

## **2. 功能模块总览**

| 模块 | 功能 |
|------|------|
| **识别** | 拍照/相册上传 → AI 识别食物 → 显示 GI 值 + 建议 |
| **食物** | 按 GI 分类筛选食物库，展示图片/名称/GI/描述 |
| **我的** | 血糖历史折线图 + 近3天统计 + 手动记录 + 历史列表 |

## **4. 页面详细需求**

---

### **4.1 【识别】页面**

#### **页面结构**
```
[ 拍照或相册选择按钮 ]
        ↓
[ 识别结果区域（识别后显示）]
        ↓
[ 底部 TabBar ]
```

#### **功能需求**

| 功能 | 描述 |
|------|------|
| **上传方式** | 支持“拍照”和“从相册选择”两种入口 |
| **AI 识别** | 调用 **通义千问 VL 模型** 进行食物识别，返回：  
  - 食物名称  
  - 置信度  
  - 对应 GI 值（从本地 GI 数据库匹配）  
  - 健康建议（基于 GI 区间） |
| **识别结果展示** |  
  - 食物图片（用户上传）  
  - 食物名称  
  - GI 值（颜色区分：绿/黄/红）  
  - 建议文案（如：适合糖尿病患者食用 / 建议少量食用 / 不建议食用） |
| **重新识别** | 提供“重新拍照”按钮 |

#### **GI 建议规则**
| GI 范围 | 标签颜色 | 建议文案 |
|--------|---------|---------|
| ≤ 55   | 绿色 | 低GI食物，适合糖尿病患者日常食用 |
| 56-69  | 黄色 | 中GI食物，建议适量搭配低GI食物 |
| ≥ 70   | 红色 | 高GI食物，不建议糖尿病患者食用 |

### **4.2 【食物】页面**

#### **页面结构**
```
[ 筛选按钮条 ]
全部  低GI  中GI  高GI
        ↓
[ 食物列表（网格布局）]
每项：图片 + 名称 + GI值 + 简述
        ↓
[ 底部 TabBar ]
```

#### **功能需求**

| 功能 | 描述 |
|------|------|
| **顶部筛选** | 单选按钮：全部 / 低GI / 中GI / 高GI |
| **本地数据筛选** | 所有食物数据存储在本地 JSON 中，筛选逻辑在前端实现 |
| **食物项展示** |  
  - 食物缩略图（本地静态资源）  
  - 食物名称  
  - GI 值（颜色标签）  
  - 简短描述（如“富含膳食纤维，升糖缓慢”） |
| **数据来源** | 内置常见食物 GI 数据库（≥50种常见食物） |

#### **示例食物数据（JSON 结构）**
```json
[
  {
    "id": 1,
    "name": "苹果",
    "gi": 36,
    "category": "low",
    "desc": "富含果胶，升糖缓慢",
    "image": "apple.jpg"
  },
  {
    "id": 2,
    "name": "白米饭",
    "gi": 83,
    "category": "high",
    "desc": "精制碳水，升糖快",
    "image": "rice.jpg"
  }
]
```

### **4.3 【我的】页面**

#### **页面结构**
```
[ 血糖历史折线图（7天/30天切换）]
        ↓
[ 近3天统计卡片 ]
平均：--   最低：--   最高：--
        ↓
[ 记录血糖按钮 ]
        ↓
[ 历史记录列表（时间倒序）]
每项：日期时间 + 血糖值 + 餐后标记
        ↓
[ 底部 TabBar ]
```

#### **功能需求**

| 功能 | 描述 |
|------|------|
| **折线图** | 使用 ECharts 或 Chart.js 绘制，展示最近 30 天的血糖记录，支持点击查看具体数值 |
| **近3天统计** | 从 `localStorage` 计算：  
  - 平均血糖  
  - 最低血糖  
  - 最高血糖 |
| **记录血糖** | 弹出表单：  
  - 血糖值（mmol/L）  
  - 测量时间（默认当前）  
  - 餐后状态（空腹 / 餐后2小时） |
| **历史记录列表** | 按时间倒序，展示所有记录，支持长按删除 |
| **数据持久化** | 所有血糖记录保存至 `localStorage`，键名：`blood_sugar_records` |

#### **localStorage 数据结构**
```json
[
  {
    "id": "2025-10-31T08:00",
    "value": 5.8,
    "time": "2025-10-31 08:00",
    "type": "fasting",  // fasting / postprandial
    "timestamp": 1730332800000
  }
]
```

## **5. 技术实现方案**

| 模块 | 技术选型 |
|------|---------|
| **前端框架** | Vue 3 + Vite + TypeScript（或纯 H5 + Vanilla JS） |
| **UI 组件** | Vant / NutUI（移动端适配） |
| **图表** | ECharts 或 Chart.js |
| **AI 识别** | 通义千问 VL 模型 API（后端代理或前端直调） |
| **数据存储** | `localStorage`（血糖记录、食物库缓存） |
| **图片上传** | `input type="file"` + `FileReader` 预览 |
| **构建工具** | Vite |
| **部署** | 静态 H5，部署至 CDN 或小程序 Webview |

## **6. 非功能性需求**

| 项目 | 要求 |
|------|------|
| **响应速度** | 拍照识别 ≤ 3秒（含网络请求） |
| **离线可用** | 食物库、血糖记录、GI 数据离线可用 |
| **兼容性** | 主流移动端浏览器（微信、Chrome、Safari） |
| **数据安全** | 血糖数据仅本地存储，不上传服务器 |
| **可扩展性** | 支持后续接入用户系统、云同步 |

## **7. 数据字典**

### **7.1 食物 GI 分类标准**

|------|--------|---------|
| 低GI | ≤ 55   | 绿色 |
| 中GI | 56-69  | 黄色 |

| 字段 | 类型 | 说明 |
|------|------|------|

## **8. 交互流程图（文字版）**

```
启动 → 底部 TabBar 显示三个页面

【识别】
拍照/相册 → 上传图片 → 调用通义千问 VL → 匹配本地 GI 库 → 显示结果

【食物】
进入页面 → 默认“全部” → 点击筛选 → 本地过滤数据 → 渲染列表

【我的】
进入页面 → 读取 localStorage → 绘制折线图 → 计算近3天统计 → 显示列表
点击“记录血糖” → 填写表单 → 保存至 localStorage → 刷新图表
```
